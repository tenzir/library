The NDJSON fixtures in this directory are synthetic test events with invented
IDs, users, domains, timestamps, and addresses.

Their field names, nesting, casing, and enum-style values are based on the
official Microsoft Graph documentation examples and resource descriptions for:

- `auditLogs/signIns`: https://learn.microsoft.com/en-us/graph/api/signin-list
- `auditLogs/directoryAudits`: https://learn.microsoft.com/en-us/graph/api/directoryaudit-list
- `security/alerts_v2`: https://learn.microsoft.com/en-us/graph/api/security-list-alerts_v2
- `security/incidents`: https://learn.microsoft.com/en-us/graph/api/security-list-incidents
- `identityProtection/riskDetections`: https://learn.microsoft.com/en-us/graph/api/riskdetection-list
- `identityProtection/riskyUsers`: https://learn.microsoft.com/en-us/graph/api/riskyuser-list
- `deviceManagement/managedDevices`: https://learn.microsoft.com/en-us/graph/api/intune-devices-manageddevice-list
- `deviceManagement/detectedApps`: https://learn.microsoft.com/en-us/graph/api/resources/intune-devices-detectedapp
- `deviceManagement/deviceCompliancePolicySettingStateSummaries`: https://learn.microsoft.com/en-us/graph/api/intune-deviceconfig-devicecompliancepolicysettingstatesummary-list
